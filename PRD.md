# 제품 요구사항 정의서 (Product Requirements Document - PRD)

**프로젝트 명**: SkillDeck (직무 역량 면접 스마트 해금 학습 서비스)  
**작성일**: 2026-05-28  
**대상 학기**: 2026년 봄학기 AI 프로그래밍 입문 (Assignment 4)  

---

## 1. 제품 개요 (Product Overview)
SkillDeck은 구직자와 대학생들이 취업 및 직무 역량 면접에 효과적으로 대비할 수 있도록 돕는 웹 기반 학습 플랫폼입니다.  
본 서비스는 전통적인 면접 대비 방식(예: 단순히 질문과 답이 모두 나열된 정적 블로그나 문서 읽기)이 주는 스포일러 문제와 지루함을 해결하고자 설계되었습니다. 모의 면접을 통해 질문 카드를 하나씩 발견하고 3D 플래시 카드로 숙달해 나가는 성장형 도감 시스템(Gamified Discovery Loop)을 제공하여, 실제 면접장과 같은 긴장감을 유지하면서 자기주도적으로 학습을 진행할 수 있도록 돕습니다.

- **대상 직무**: 사무/행정, 기계, 전기, 전자, ICT/SW, 화학 (6대 전공 도메인) 및 공통 면접 분야
- **핵심 가치**: 
  - 답을 미리 보지 않고 면접을 마주하는 실전 긴장감 제공
  - 직무별 전공 학습 진척도의 실시간 정량적 시각화
  - 개별 오답 및 중요 지식을 체계적으로 복습하는 자기주도 학습 환경

---

## 2. 타겟 사용자 (Target Users)
SkillDeck의 타겟 사용자는 전공 면접 및 직무 인성 면접을 앞두고 있는 취업 준비생과 대학생입니다.

### 사용자 페르소나 및 요구사항(Pain Points)
1. **취업 준비생 (A씨, 26세, ICT/SW 지원자)**
   - *문제점*: 기술 면접을 준비할 때 책이나 족보를 보면 답안이 바로 옆에 적혀 있어 실제로 내가 아는 것인지, 그냥 눈으로 읽고 넘어가는 것인지 파악하기 어려움.
   - *요구사항*: 답안이 가려진 상태에서 질문을 마주하고 스스로 인출(Recall)해보는 실전형 테스트가 필요함.
2. **비전공 전직 희망자 (B씨, 29세, 전기 분야 지원자)**
   - *문제점*: 전공 기초 지식과 실무 돌발 사고 대응 요령이 머릿속에 파편화되어 있어 어떤 부분을 집중적으로 복습해야 하는지 정량화하기 어려움.
   - *요구사항*: 나의 분야별 질문 학습 완성도(발견율, 숙달율)를 백분율로 직관적으로 확인하고 오답을 따로 모아보고 싶음.

---

## 3. 프로젝트 목표 (Project Goals)
- 사용자가 데이터 중심의 취업 준비 진척도를 실시간으로 확인할 수 있는 **완전 클라이언트 사이드 단독 구동형 웹 서비스** 구현.
- 직무 분야 간 진행도 간섭이 없도록 데이터 모델을 격리하고, 공통 분야 지식과 전공 분야 지식의 융합 테스트 환경 제공.
- 웹 접근성과 미려한 글래스모피즘(Glassmorphism) 테마 스타일을 준수하여 시각적인 프리미엄 디자인 완성.
- 별도의 백엔드 설치 없이 브라우저의 `localStorage`만을 이용해 사용자의 북마크(중요 노트) 및 커스텀 추가 카드, 카드의 발견/숙달 상태를 안전하게 영구 캐싱.

## 4. 핵심 사용자 시나리오 (Core User Scenario)
1. **직무 선택 및 대시보드 확인**: 사용자는 웹사이트에 접속해 상단 드롭다운에서 자신의 목표 직군(예: `ICT/SW`)을 선택합니다. 대시보드의 진행률(질문 발견 진행도, 숙달 지수)이 모두 0%인 빈 도감 상태를 확인합니다.
2. **모의 면접 수행**: [모의 면접] 탭으로 이동해 '테스트 시작하기'를 클릭합니다. 45초의 타이머가 흐르는 동안 화면에 출제된 무작위 질문(SW 전공 4문항 + 무작위 순서의 공통 질문 1문항)에 구두로 답변을 시도합니다.
3. **자가 진단 및 카드 발견**: '모범 해답 보기'를 누르면 상세 가이드가 표시됩니다. 스스로 답변의 완성도에 따라 난이도 버튼(`쉬움/유창함`, `보통/애매함`, `어려움/모름`) 중 하나를 클릭합니다. 이때 출제된 질문 카드들은 도감에 자동으로 '발견(해금)' 처리됩니다.
4. **학습된 카드 도감 확인 및 중요 표시**: [학습된 카드] 탭으로 이동하면 모의 면접에서 해금된 질문 카드들이 3D 카드 형태로 나열되어 있습니다. 마우스를 올리거나 클릭하여 카드를 뒤집어가며 복습하고, 자주 틀리는 카드는 우측 상단 별표(★)를 눌러 북마크합니다.
5. **중요 노트(오답 노트) 집중 복습**: [중요 노트] 탭으로 이동하여 자신이 별표 표시한 카드와 스스로 작성한 커스텀 카드들만 선택된 직무별로 모아서 반복 복습하고 메인 대시보드로 돌아가 누적 진행 게이지(질문 발견율, 숙달율 %)가 상승한 모습을 확인합니다.

## 6. 페이지 구조 (Page Structure)
SkillDeck은 단일 페이지 어플리케이션(SPA) 구조로 디자인되었으며, 좌측 내비게이션 메뉴를 통해 아래 4가지 핵심 뷰(Section)가 유기적으로 연동됩니다.

1. **대시보드 뷰 (Dashboard View)**: 직무별 진척 게이지, 종합 평가 대시보드 위젯, 오늘의 무작위 발견 질문 추천 카드 배치.
2. **학습된 카드 뷰 (Study Deck View)**: 해금된 카드의 전체 목록 그리드, 키워드 검색바, 상태 필터 그룹, 커스텀 카드 추가 모달 트리거 배치.
3. **중요 노트 뷰 (Bookmarks View)**: 별표(★) 북마크 카드와 사용자가 추가한 커스텀 카드가 해당 분야별로 정렬되어 표시되는 아카이브 공간.
4. **모의 면접 뷰 (Mock Quiz View)**: 테스트 시작 대기 화면(공통 분야 진입 제한 경고 포함), 활성 퀴즈 아레나(질문 카드, 45초 타이머, 힌트 보기, 자가 진단 평가 버튼), 최종 성과표 피드백 및 결과 분석 패널로 구성.

## 7. 기술적 요구사항 (Technical Requirements)
- **개발 언어**: Vanilla HTML5 (시맨틱 마크업), CSS3 (Modern Flexbox, Grid, Custom Variables), Javascript (ES6+, LocalStorage API)
- **외부 라이브러리**: 종속성 최소화를 위해 별도의 외부 프레임워크(React, Vue 등)나 빌드 도구(Webpack, Vite)를 배제하고 브라우저가 다이렉트로 해석 가능한 순수 클라이언트 파일로만 작성.
- **상태 보존**: 브라우저를 새로고침하거나 창을 닫아도 복습 이력과 발견 상태가 영구 유지되도록 `localStorage`와 실시간 직렬화 동기화.
- **배포 플랫폼**: Vercel을 이용해 GitHub 레포지토리와 연동하여 배포를 자동화(Continuous Deployment)하고 퍼블릭 URL 제공.

---

## 8. 디자인 요구사항 (Design Requirements)
- **비주얼 스타일**: 프리미엄 테마 구현을 위해 반투명 아크릴 느낌을 주는 **글래스모피즘(Glassmorphism)** 효과 적극 적용 (`backdrop-filter: blur`).
- **도메인별 포인트 컬러 (Theme Palette)**:
  - 사무/행정: Indigo Purple (`#6366f1`)
  - 기계: Crimson Amber (`#f59e0b`)
  - 전기: Electric Yellow (`#eab308`)
  - 전자: Cyan Teal (`#14b8a6`)
  - ICT/SW: Electric Blue (`#3b82f6`)
  - 화학: Emerald Green (`#10b981`)
  - 공통 분야: Slate Neutral (`#64748b`)
- **인터랙션 및 모션**:
  - 카드 호버 시 위로 살짝 뜨는 마이크로 상호작용 및 그림자 번짐 효과.
  - 플래시 카드 플립 시 부드러운 회전 각도를 나타내는 `transform: rotateY(180deg)` 및 `perspective` 3D 공간 왜곡 렌더링.
  - 반응형 웹 레이아웃 지원으로 데스크톱, 태블릿, 모바일 기기 화면 비율에 맞추어 사이드바와 카드 그리드 수가 유연하게 변하는 Responsive Layout 제공.
