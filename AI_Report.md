# AI 협업 개발 및 배포 보고서 (AI-Assisted Development Report)

본 보고서는 **SkillDeck** 웹 서비스 개발 과정에서 AI 코딩 어시스턴트와의 협업 경험을 정리한 문서입니다. 과제 평가 기준(Step 3)에 맞추어 도구 사용처, 주요 프롬프트, 수정 사항 및 트러블슈팅 사례를 상세히 서술합니다.

---

## 1. 사용한 AI 도구 (AI Tools Used)
- **Antigravity AI (Gemini 기반)**: 프로젝트 기획서(PRD) 초안 작성, 전체 파일 구조 기획 및 파일 자동화 생성 수행.
- **Gemini 3.5 Flash**: 직무별 전공 지식 면접 데이터셋(70개 질문/답안) 정제 및 HTML/CSS/JS 상세 디버깅.

---

## 2. AI 협업 및 분담 역할 (Tasks Handled by AI)
1. **아이디어 브레인스토밍 및 구체화**: 취업용 면접 대비 서비스에서 한 단계 나아가 특정 학업/전공 지식을 점검하는 '직무 역량 면접 스마트 카드 학습 툴(SkillDeck)'로 포지셔닝 변경.
2. **성장형 도감 해금 루프(Gamified Discovery Loop) 기획**: 단순히 모든 지식이 노출된 상태가 아닌, **[모의 면접]**을 치를 때마다 새로운 질문이 **[학습된 카드]**에 한 장씩 발견(해금)되어 축적되는 성장형 게임 형태의 아키텍처 구현.
3. **대규모 전공 데이터셋 및 공통 면접 생성**: 사무/행정, 기계, 전기, 전자, ICT/SW, 화학의 6대 분야 전공 기술 핵심 기출 질문(60개)에 추가로 **인성/가치관 공통 질문 10개**를 결합하여 총 70개의 대규모 DB를 구축.
4. **수학적 게이지 계산 및 공통 분야 격리**: 직무를 변경할 때마다 포인트 테마색이 달라지며, 절대 개수를 숨기고 **백분율(발견율 %, 숙달율 %)**로만 직무 준비 수준을 대시보드에 뿌려주는 연산 로직 구현. 특히 공통 질문이 해금되었을 때 전공 직무 진행도가 함께 오르는 간섭 문제를 해결하기 위해 공통 분야의 진행률 트래킹과 도감을 완전히 전공과 분리 설계함.

## 3. 대표적인 협업 프롬프트 (Representative Prompts Used)

### 프롬프트 1: 발견/해금식 암기 학습 루프 아이디어 구체화
> **"사용자가 미리 모든 질문 카드를 다 볼 수 있다면 시험의 스포일러가 되어 긴장감이 떨어진다고 생각해. 처음에는 '학습된 카드' 목록을 0개로 비워두고, 모의 면접(5문항 퀴즈)을 볼 때 나온 질문들이 해금되어 공부할 수 있게 쌓이는 '도감 해금형' 학습 시스템을 설계하고 싶어. 이 조건에 맞는 JavaScript 상태 관리와 로컬 스토리지 동기화 전략을 알려줘."**

### 프롬프트 2: 공통 질문 무작위 삽입 및 순서 랜덤화 로직
> **"모의 면접을 시작할 때, 사용자가 선택한 직무 데이터에서 4문제를 무작위로 뽑고, 공통 질문 데이터셋(10개)에서 1문제를 무작위로 뽑아서 총 5문제를 출제하고 싶어. 이때 공통 질문이 5문제 중에서 몇 번째 순서(1st~5th)로 나올지도 실행할 때마다 랜덤으로 섞이게 만드는 깔끔한 JS Array 조작 코드를 짜 줘."**

### 프롬프트 3: 대시보드 수치 숨기기 및 백분율 변환
> **"전체 질문이 총 몇 개인지(예: 0 / 60)를 밝히는 대신 진행률을 오직 100% 기준으로만 시각화하고 싶어. 활성화된 직무 카테고리에서 최대 발견 가능한 개수를 20개(직무 10개 + 공통 10개)로 놓고 현재까지 발견된 카드의 비율을 계산하여 %로만 트래킹되도록 대시보드 함수를 업데이트해줘."**

## 4. AI 생성 코드 수정 및 개선 사항 (Modifications & Improvements)

1. **이벤트 전파(Event Propagation) 버그 방지**:
   - *문제*: AI가 작성한 초기 카드 뒤집기 코드에서는 카드 뒷면에 위치한 북마크(★) 클릭이나 난이도 버튼(`😀`, `😐`, `😰`)을 누를 때마다 부모 컨테이너로 클릭 이벤트가 전파되어 카드가 본래대로 다시 뒤집히는 불쾌한 UX가 발생했습니다.
   - *해결*: JavaScript 코드 내 이벤트 핸들러에 `e.stopPropagation()`을 삽입하여 버튼 영역 클릭 시 카드 뒤집기 동작이 간섭받지 않고 본래의 비즈니스 로직(북마크 저장, 평점 기록)만 처리되도록 수정했습니다.

2. **공통 질문(Common) 테마 상속**:
   - *문제*: 공통 질문은 카테고리가 `common`이어서 직무별 전용 클래스(`theme-software` 등)가 적용되지 않고 스타일시트의 기본 색상만 적용되어 시각적으로 분리되어 보였습니다.
   - *해결*: 공통 카드가 도감(학습된 카드)에 렌더링될 때는 개별 카드 요소에 `theme-common` 클래스가 부여되어, 현재 사용자가 보고 있는 도메인의 포인트 컬러를 자연스럽게 상속받아 시각적 이질감이 없도록 CSS 변수를 연결하였습니다.

3. **로컬 스토리지 키 동기화**:
   - *문제*: 여러 페이지에서 데이터를 서로 다르게 쓰고 읽어 초기화 시 값이 덮어씌워지는 오류가 예상되었습니다.
   - *해결*: `app.js` 단일 파일로 동적 렌더링을 완전히 제어하고, `getAllCards()` 함수를 경유하여 언제나 `dataset.js` 데이터와 `localStorage` 내 커스텀 배열을 실시간 병합하여 읽도록 설계 통합을 이뤄냈습니다.

4. **공통 분야 진척도 간섭 분리 및 예외 처리**:
   - *문제*: 공통 질문 해금 시 모든 전공 분야의 진행도(발견율)가 일괄 동반 상승하는 결함이 있었습니다.
   - *해결*: 전공 진행률 모수 계산에서 공통 카드를 배제하고, 공통 분야 도감 및 진행 바를 대시보드에 별도 항목으로 분리시켰습니다. 추가로 공통 분야 선택 시 모의 면접 시험을 단독으로 볼 수 없도록 경고 노출 및 시작 버튼을 비활성화하는 예외 방어 코드를 작성했습니다.
